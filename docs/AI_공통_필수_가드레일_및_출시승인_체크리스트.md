# AI 공통 필수 가드레일 및 출시승인 체크리스트

이 문서는 AI 기능 공통 보안/권한/근거/운영 기준의 단일 기준 문서입니다.

## 적용 대상 기능

- 1번: 첫 화면 AI 업무 도우미
- 3번: AI 요약 카드
- 4번: 다음 액션 추천
- 5번: 작성 보조
- 6번: 이상 징후 알림
- 7번: 관리자 운영 코파일럿

## 1) 민감정보 마스킹 (입력/출력/로그 전 구간)

1. 주민등록번호, 계좌번호, 카드번호, 전화번호, 이메일, 주소, 인증 토큰, 세션 ID, API Key는 마스킹 처리한다.
2. 모델 프롬프트에는 원문 민감정보를 직접 전달하지 않는다.
3. 로그/분석 저장소에는 해시 또는 부분 마스킹 값만 저장한다.
4. 첨부파일/자유입력 텍스트 내 민감정보도 동일 정책으로 탐지/차단한다.

## 2) 권한 범위 밖 데이터 응답 금지

1. 사용자/조직/역할 기반 접근제어를 모델 호출 전에 서버에서 강제 검증한다.
2. 권한 없는 데이터는 조회 자체를 차단하고, 응답은 `권한 없음` 메시지로 통일한다.
3. 모델이 임의 추론으로 타 조직/타 허브 데이터를 생성하지 못하도록 컨텍스트를 최소화한다.
4. 관리자 기능도 최소권한 원칙을 적용하고, 예외 권한은 사유와 만료일을 기록한다.

## 3) 답변 근거 출처 표시

1. 모든 AI 응답에는 근거 출처를 함께 표시한다.
2. 출처에는 데이터 종류, 생성 시각, 조회 범위, 필터 조건을 포함한다.
3. 사용자는 클릭으로 원문 화면(해당 허브/목록/상세)으로 이동 가능해야 한다.
4. 근거가 불충분한 답변은 `추정`으로 명시하고 자동 실행 액션을 금지한다.

## 4) 오답 신고 버튼 및 로그 추적

1. 모든 AI 카드/응답 컴포넌트에 오답 신고 버튼을 제공한다.
2. 신고 시 아래 항목을 구조화 저장한다.
   - 사용자 역할
   - 화면
   - 질문
   - 답변
   - 근거
   - 모델 버전
   - 요청 ID
   - 시간
   - 신고 사유
3. 신고 접수 후 운영자가 상태(`접수`/`분석중`/`조치완료`)를 관리할 수 있어야 한다.
4. 반복 오답 유형은 규칙/프롬프트/권한 필터 개선 항목으로 자동 등록한다.

## 5) 운영 필수 정책

1. 기본 정책은 거부(`Default Deny`)이며, 허용된 범위만 응답한다.
2. AI가 제안한 액션은 사용자 최종 확인 없이 실행하지 않는다.
3. 민감정보 포함 가능 기능은 테스트 계정/테스트 데이터로만 검증한다.
4. 응답 품질 지표(정확도, 재현율, 신고율, 수정 리드타임)를 주 단위로 점검한다.

## 6) 출시 승인 체크리스트

- [ ] 마스킹 테스트 통과
- [ ] 권한 우회 테스트 통과
- [ ] 출처 표시 UI/링크 동작 확인
- [ ] 오답 신고 저장 및 추적 대시보드 확인
- [ ] 감사 로그 누락 없음 확인
- [ ] 롤백 절차 문서화 완료

## 7) 구현 점검 포인트 (개발자용)

1. 프롬프트 구성 전 단계에서 민감정보 redaction을 강제한다.
2. 모델 호출 직전 서버에서 조직/역할/리소스 범위를 재검증한다.
3. 모델 응답 후 클라이언트 렌더 직전에 2차 마스킹 검사를 수행한다.
4. 출처 메타데이터가 없는 응답은 렌더링하지 않거나 `추정` 라벨을 강제한다.
5. 자동 실행 경로가 연결되어 있으면 차단하고 확인 다이얼로그를 의무화한다.
6. 오답 신고는 운영 큐와 감사로그에 동시에 적재한다.
