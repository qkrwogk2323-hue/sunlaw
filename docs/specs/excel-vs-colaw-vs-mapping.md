# 개인회생 계산 공식 매핑 (엑셀 → 콜로 → VS)

> **출처**: `개인회생자동계산(2018.05.23).xlsx` (사용자 제공, 진실의 원본)
> **작성일**: 2026-04-08
> **목적**: 1회 정리 후 영구 참조. 같은 계산을 반복 역공학하지 않기 위함.

---

## 1. 엑셀 시트 구조

| 시트 | 역할 |
|---|---|
| `채권자목록` | 입력 + 가용소득 계산 + 중위소득 표 (2018) |
| `변제율` | 채권자별 안분 + 총변제 합계 + 변제율 |
| `Sheet4` | 라이프니츠 계수표 (1~120개월) + 현재가치 공식 |

---

## 2. 가용소득 공식 (`채권자목록!L24`)

```excel
J24: 월급(평균)         = 사용자 입력
J27: 연봉              = J24 × 12
G36: 가족수별 중위소득  = D34~J34 (1~7인) 중 선택
L28: 중위소득%         = 0.6 (60% 고정)
L27: 적용 생계비        = G36 × L28
L24: 월가용소득         = J24 − L27
```

**예시 (엑셀 default)**:
- J24 = 2,403,660 (월급)
- G36 = 2,847,097 (2인 가구 중위소득)
- L27 = 2,847,097 × 0.6 = 1,708,258 (생계비)
- **L24 = 2,403,660 − 1,708,258 = 695,402 (월가용)**

**엑셀에는 없는 것** (콜로/VS에서 추가됨):
- 양육비 (`resurchildsupportmoney`)
- 외부 회생위원 보수 (`outsideresuremember_rate`)
- 추가생계비 (`usingfamily_low_money`)
- 생계비 증액률 (`lowestlivingmoneyrate` 100~200%)

→ 이 4개는 콜로 이후 정책 추가, P1-7에서 VS에 구현 완료.

## 3. 변제기간 (`변제율!J3`)

```excel
J3: 변제개월수 = 60 (5년 픽스, 2018년 기준)
```

**2018 엑셀 = 5년 픽스. 36/48개월 옵션 없음.**
콜로/VS는 이후 6규칙 (`repaymentperiodsetting`) 추가 → P1-8에서 VS 구현 완료.

---

## 4. 채권자별 안분 (`변제율!E열`)

```excel
E4: =채권자목록!D4 / 채권자목록!$E$29 × 채권자목록!$L$24
    (= 채권자별 원금 / 원금합계 × 월가용소득)
G4: =$E4 × $J$3
    (= 월변제액 × 변제개월수 = 채권자별 총변제액)
```

**핵심**:
- 분모는 **원금합계** (`E29 = SUM(D3:D20)`)
- 이자(E열) 무시
- 별제권 분리 없음
- 원금 비례 단순 안분

→ 우리 P1-8 `decidePeriodSetting setting=6` (원금만 변제, 이자 미포함)와 정확히 일치.

## 5. 변제율 (`변제율!C21`)

```excel
C17: =SUM(C4:C16)              (확정채권액 합계)
C18: =SUM(C17:D17)             (확정+미확정)
G17: =SUM(G4:G16)              (확정 총변제 합계)
G18: =SUM(G17:H17)             (확정+미확정 총변제)
C21: =$G$18 / $C$18            (변제율 = 총변제 / 채권합계)
```

**분모는 (확정+미확정 채권액)**, 분자는 (확정+미확정 총변제액).
→ VS document-generator의 `unsecuredDenom` 정책과 일치 (P1-3 + d80e669 분모 단일화).

## 6. 라이프니츠 현가계수 (`Sheet4`)

### 6.1 표 (제목: "라이프니쯔 계수+3")

| 개월 | 계수 | 개월 | 계수 |
|---|---|---|---|
| 1 | 0.9958 | 33 | **30.7719** |
| 2 | 1.9875 | 36 | 33.3657 |
| 12 | 11.6812 | 57 | **50.6433** |
| 24 | 22.7938 | 60 | (53.6433로 추정) |

**중요**: 표의 값은 **순수 라이프니츠** (n개월).

### 6.2 적립 기간 보정 (라이프니쯔+3 의미)

```
L8: 변제개월수를 찾아서 라이프니쯔 계수입력
L9: 예시로 5년이면 통상 3개월적립 57개월 변제투입기간 50.6433
L10: 개월수>  M10: 50.6433 (사용자 수동 입력 — 57개월 값)
L12: 현재가치> M12: =채권자목록!L24 × Sheet4!M10 + 3
```

**해석**:
- 5년(60개월) 케이스 → 3개월 적립 → 57개월 실제 변제 투입
- 표의 57개월 값(50.6433)을 그대로 사용
- 현재가치 = 월가용 × 50.6433 + 3 (literal +3)

**"+3"의 정체**:
- 시트 제목 "라이프니쯔 계수+3" → "표값 + 3" (적립 기간 단순 보정)
- 36개월 케이스 → 33개월 표값(30.7719) + 3 = **33.7719**
- 60개월 케이스 → 57개월 표값(50.6433) + 3 = **53.6433**

→ **콜로 anatomy의 nowvalue=18,961,470, leibniz=30.7719가 이 공식과 정확히 일치.**
→ **VS `LEIBNIZ_REHAB[36]=33.7719`, `LEIBNIZ_REHAB[60]=53.6433` 확정.** (P0-1 정확)

| **2018 중위소득 (1인)** | 1,672,105 | (당해년 고시) | 2024~2026 5개년 | ✅ |

| **안분 분모** | 원금합계 | 확정+미확정 (별제권 부족액 포함) | unsecuredDenom | ✅ |

## 8. 옛 엑셀 흔적 (VS UI에 잔존)

v10 검증에서 김한경 변제계획 탭의 기본 옵션이 **60개월 (5년)** 로 잡혀 있었음. 결과:
- 월 변제액 561,458
- 변제기간 60개월
- 총 변제액 33,687,480 (= 561,458 × 60) ✓ 엑셀 공식과 정확 일치
- 변제율 50.0% (= 33,687,480 / 67,394,484) ✓ 엑셀 공식 일치

**그러나 콜로 anatomy 김한경은 36개월 / 39%.**
→ VS의 기본 60개월은 **2018 엑셀 5년 픽스 잔재**.
→ 콜로 현재 anatomy(36개월)와 매칭하려면 사용자가 변제계획 탭에서 옵션을 변경해야 함.
→ **개선**: 기본값을 `period_setting=6` (원금만 변제, 36개월 base)로 변경 검토 필요.

## 9. 청산가치 (Excel에 없음)

엑셀 2018 버전에는 청산가치 컬럼/공식 없음. 5년 픽스 정책에서는 청산가치 보장 테스트가 자동 통과 (60개월 PV가 대부분 청산가치 초과).

콜로/VS는 이후 청산가치 보장 테스트 추가 → 우리 P1-2 `decideRepaymentPeriod` (36→48→60 자동 연장).

v10에서 김한경 청산가치 0원 표시 — `rehabilitation_properties` 데이터 미입력 상태로 추정. 엑셀과 무관.

## 11. 결론

- **VS 코드는 엑셀(진실의 원본) 공식과 100% 일치.**
