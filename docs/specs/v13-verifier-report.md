## 3. 🚨 신규 발견 P0 — 가용소득 계산 불일치

| | 월소득 | 차감 생계비 | 가용소득 |
|---|---:|---:|---:|
| **colaw** | 2,100,000 | 1,025,695 | **1,074,305** |
| **VS** | 2,100,000 | 1,538,543 | **561,458** |
| **차이** | — | +512,848 | **−512,847 (−47.7%)** |

**원인 추정**: VS는 생계비로 중위소득 **100%**를 차감하지만, colaw는 약 **67% (≈ 2/3)**를 차감. 법정 기준은 일반적으로 중위소득 60%이며, colaw가 진실원본에 가까움.

**파급 효과**:
- VS 변제율 64.6%는 가용소득이 절반이라 산출된 결과. colaw 기준 가용소득 1,074,305를 적용하면 60개월 총변제액 = 64,458,300 → 변제율 약 **123.6%** (즉 capital60 옵션은 100% 초과 만기 도달 → 36개월 옵션이 적정)
- 실제 colaw 진실원본은 36개월 강제(`forcingrepaymentmonth=36`) + 가용소득 1,074,305 → 총변제 38,675,000 → 분모 C(52,147,802) → **74.2%**
- 또는 colaw anatomy 39%는 다른 기준 (totalDebt 67,394,484 분모 → 38,675,000/67,394,484 = 57.4%) — 여전히 불일치
- **재검토 필요**: colaw 39% 산출식 자체가 36개월 × 약 730,000 = 26,280,000 / 67,394,484 = 39% — 가용소득이 730,000일 가능성. 즉 colaw 내부적으로 한 번 더 차감(보험료/관리비 등)

→ 결론: **두 시스템 모두 가용소득 산출식에 진실원본 격차 있음**. P0 분모 변경만으로 colaw 일치 불가. 가용소득 산식 자체를 병렬 검증 필요.
