# P0-3 수정본 — 확정/미확정 채권 구조 재정의

**날짜**: 2026-04-07
**사유**: colaw 핸들러 소스 분석 결과, 미확정 채권은 단순 status flag가 아니라 **별제권부 담보 부족액** 또는 **기타 미확정(신탁재산 등)** 두 경로로 구성됨. 원본 P0-3의 `confirmation_status text` 컬럼 방식은 폐기.

---

## 배경 (요약)

- colaw `popupRescureCreditorList` 핸들러:
  - `.layer-attached-documents` → `popupAttachedDocumentsList` (별제권부 부속서류 1~4)
  - `.layer-unconfirmed-bond` → `popupUnConfirmedBondView` (기타 미확정)
  - 두 경로는 **상호 배타** (핸들러 주석 확인)
- 김한경 제이비우리캐피탈 자동차담보대출금:
  - 총액 23,835,499 (capital)
  - 담보평가 10,680,000 (`dambosum`, `frm_bond_calculate`)
  - 부족액 13,155,499 = B_29의 "미확정" 컬럼과 원 단위 일치
- `unsettlementbond` 체크박스는 per-creditor 플래그지만 김한경 10건 모두 `false` → 이 케이스의 미확정은 **경로 A(별제권부)에서 자동 도출**된 것

---

## P0-3 변경사항

### (a) migration 변경

```sql
-- 원본 P0-3 폐기
-- alter table rehabilitation_creditors add column confirmation_status text ...

-- 수정본
alter table rehabilitation_creditors
  add column if not exists is_secured boolean not null default false,
  add column if not exists secured_collateral_value numeric(18,0) not null default 0,
  add column if not exists is_other_unconfirmed boolean not null default false;

-- 제약: 별제권부와 기타 미확정은 상호 배타
alter table rehabilitation_creditors
  add constraint creditors_secured_xor_unconfirmed
  check (not (is_secured and is_other_unconfirmed));

-- 별제권부는 담보평가액 > 0 이어야 의미가 있음
alter table rehabilitation_creditors
  add constraint creditors_secured_requires_collateral
  check (not is_secured or secured_collateral_value > 0);
```

### (b) 도출 함수 (새 파일 `src/lib/rehabilitation/creditor-classification.ts`)

```ts
export interface CreditorClassification {
  confirmedAmount: number;    // 확정 무담보 금액
  unconfirmedAmount: number;  // 미확정 금액 (별제권부 부족액 또는 기타)
  securedCoveredAmount: number; // 담보로 충당되는 부분 (변제계획 제외)
}

export function classifyCreditor(c: {
  total_claim: number;
  is_secured: boolean;
  secured_collateral_value: number;
  is_other_unconfirmed: boolean;
}): CreditorClassification {
  if (c.is_other_unconfirmed) {
    return {
      confirmedAmount: 0,
      unconfirmedAmount: c.total_claim,
      securedCoveredAmount: 0,
    };
  }
  if (c.is_secured) {
    const covered = Math.min(c.secured_collateral_value, c.total_claim);
    const deficiency = Math.max(0, c.total_claim - covered);
    return {
      confirmedAmount: 0,
      unconfirmedAmount: deficiency,
      securedCoveredAmount: covered,
    };
  }
  return {
    confirmedAmount: c.total_claim,
    unconfirmedAmount: 0,
    securedCoveredAmount: 0,
  };
}
```

### (c) schedule-generator 변경

`tieredTaxPriority` 모드 내부에서 각 채권자를 `classifyCreditor`로 분류하고,

- `securedCoveredAmount`: 변제계획 원금 배분에서 제외 (별제권으로 회수)
- `confirmedAmount` + `unconfirmedAmount`: 무담보 pro-rata 대상
- 출력 시 회차별 `confirmed_principal` / `unconfirmed_principal` 두 컬럼 분리

### (d) 검증 체크포인트 (김한경 기준)

```ts
const creditor = { // 제이비우리캐피탈(주)
  total_claim: 23_835_499,
  is_secured: true,
  secured_collateral_value: 10_680_000,
  is_other_unconfirmed: false,
};
const c = classifyCreditor(creditor);
expect(c.securedCoveredAmount).toBe(10_680_000);
expect(c.unconfirmedAmount).toBe(13_155_499);
expect(c.confirmedAmount).toBe(0);
```

월별 미확정 합계가 B_29 page 13 요약값과 원 단위 일치해야 함 (± 라운딩 1원 허용).

### (e) migration 스크립트 (추후)

`scripts/colaw-migration/migrate-colaw-to-vs.ts`의 채권자 수집 블록 확장 필요 — 현재 capture는 `capital`, `classify` 등만 잡고 담보평가는 미수집. colaw `popupAttachedDocumentsList` 모달에서 채권별 담보평가액을 추출해야 하며, 필드명은 모달을 정상 플로우로 열어 확인 필요 (현재 직접 POST는 세션 리다이렉트 + 서버 500으로 차단).

임시 마이그레이션 전략: `dambosum` (케이스 헤더)을 "is_secured=true + 자동차담보/부동산담보 cause code를 가진 단일 채권자"에 귀속. 다중 담보 케이스는 필드명 확정 후 재마이그레이션.
