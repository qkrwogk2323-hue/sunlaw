# 버그 및 오류로그 레지스터

이 문서는 현재 저장소에서 확인된 주요 버그, 설명 가능한 오류 코드, 운영 중 반드시 추적해야 하는 사고 이력을 남기는 레지스터다.

용도는 아래와 같다.

1. 이미 발생했던 사고를 재발 방지 대상으로 남긴다.
2. `UNEXPECTED_EXCEPTION` 같은 설명되지 않은 예외를 버그 후보로 누적한다.
3. 코드형 오류가 어떤 영역에 존재하는지 빠르게 찾는다.

## 1. 기록 원칙

- 설명 가능한 업무 오류는 `오류 코드`로 기록한다.
- 설명되지 않은 예외는 `버그 후보`로 기록한다.
- 이미 수정된 사고도 삭제하지 않고 `해결됨`으로 남긴다.
- 화면 상태, 원인, 영향 범위, 수정 커밋 또는 관련 소스를 함께 남긴다.

## 2-1. 이번 라운드에서 반드시 기억할 핵심 버그

아래 3개는 별도 우선 추적 대상으로 유지한다.

1. 플랫폼 조직이 조직 신청 확인을 못 받던 버그
   - 현재 상태: 해결됨
   - 관련 항목: `BUG-20260322-003`
2. 플랫폼 조직 메뉴엔 없던 대시보드가 기본 진입처럼 보이던 버그
   - 현재 상태: 진행중
   - 관련 항목: `BUG-20260322-005`
3. 중요한 기록과 감사로그를 볼 수 있는 경로가 없던 버그
   - 현재 상태: 진행중
   - 관련 항목: `BUG-20260322-006`

## 2-2. 버그 기록 최소 완성 조건

아래 항목이 모두 있어야 버그 기록을 완료로 본다.

1. 증상
2. 직접 원인
3. 근본 원인
4. 원인 해결 여부
5. 임시 조치 또는 영구 조치
6. 관련 위치

하나라도 비어 있으면 미완성 버그 기록으로 본다.

## 3. 설명되지 않은 예외 관리

아래는 반드시 버그 후보로 취급한다.

- `UNEXPECTED_EXCEPTION`
- 화면 설명 없이 전역 예외 화면으로 떨어지는 경우
- provider 누락으로 인한 런타임 크래시
- 일부 데이터 실패가 전체 페이지 크래시로 번지는 경우
- 현재 메뉴와 다른 기본 진입 화면으로 이동하는 경우

## 5. 기존 오류로그 기록 방법

새 버그나 오류를 확인하면 아래 두 곳을 같이 갱신한다.

1. `docs/버그리포트_운영_작성가이드.md`
   - 개별 버그 문서 작성 기준
2. `docs/버그_및_오류로그_레지스터.md`
   - 현재 사고 이력과 오류 코드 레지스터 누적

## 6. 전수 조사 시 확인 순서

1. 이 기능은 어느 소속인가
2. 지금 어느 메뉴에 노출되는가
3. 기본 진입 경로가 메뉴 첫 주소와 같은가
4. 상태 화면과 과거 로그가 모두 보이는가
5. 설명 가능한 오류는 코드와 원인으로 표시되는가
6. 설명되지 않은 예외는 `버그 후보`로 레지스터에 올라와 있는가

## 7. 다음 정리 대상

아래 항목은 계속 전수 조사한다.

- 플랫폼 조직 기본 진입 경로와 메뉴 단일 원천 통일
