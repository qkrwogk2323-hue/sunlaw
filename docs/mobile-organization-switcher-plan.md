# 모바일 조직 전환 UX 및 사건허브 연동 계획

## 목표
모바일에서 현재 조직을 항상 명확히 표시하고, 한 손 조작으로 조직을 전환할 수 있어야 한다.
조직 전환 직후 사건허브, 사건목록, 문서, 일정은 새 조직 문맥으로 즉시 재계산되어야 한다.

## 필수 요구사항
1. 현재 조직은 모바일 헤더 상단에 항상 표시한다.
2. 조직 전환 UI는 Bottom Sheet 또는 Full-screen Sheet로 제공한다.
3. 조직 목록이 8개 이상이면 검색창을 상단에 고정한다.
4. 최근 사용 조직 3개를 최상단에 노출한다.
5. 현재 조직은 체크 아이콘과 강조 배경으로 표시한다.
6. 조직 전환은 로컬 상태만 바꾸지 말고 URL 기반 문맥 전환으로 처리한다.
7. 조직 전환 후 서버 권한 재검증을 수행한다.
8. 현재 페이지 문맥은 가능한 한 유지한다.
9. 사건허브 접근 권한이 없는 경우 허브 목록으로 안전하게 이동한다.
10. 전환 중 중복 탭을 막기 위해 버튼을 비활성화한다.

## 권장 라우팅
- `/org/[organizationSlug]/cases`
- `/org/[organizationSlug]/clients`
- `/org/[organizationSlug]/hub`

## 구현 대상

- `organization-switch-sheet.tsx`

- 조직 전환 route helper 또는 server action
- 헤더 또는 `mode-aware-nav.tsx` 진입점 연결

## 사건허브 연동 규칙
1. 조직 전환 후 허브 연동 상태를 다시 계산한다.
2. 협업 인원, 열람 인원, 최근 활동은 새 조직 기준으로 재조회한다.
3. 같은 사건이라도 조직별 권한 차이를 반영한다.

## 완료 기준
1. 모바일에서 현재 조직이 항상 보인다.
2. 조직 전환 후 현재 페이지 문맥이 유지된다.
3. 사건허브가 조직 기준으로 즉시 갱신된다.
4. 타입 체크와 관련 e2e가 통과한다.
