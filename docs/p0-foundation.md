# P0 Foundation Summary

## 핵심 원칙

1. 사건은 다중 조직 참여형 협업 객체다.
2. Billing은 모든 사건의 공통 기본 기능이다.
3. Collection은 추심 사건 선택 모듈이며, 내부 운영용 별도 Workspace다.
4. 추심 보수는 Billing이 아니라 Collection Compensation 도메인이다.
5. 조직은 법인의 하위부서가 아니라 독립 운영 세포다.
6. 사건 상세는 공통 Case Shell을 사용한다.

## 공통 Case Shell

1. Overview
2. Communication
3. Documents
4. Schedule
5. Participants
6. Billing
7. Timeline

## 선택 모듈

1. Collection
2. Insolvency
3. Settlement
