# 마이그레이션 스키마 설명서 (0001~0070)

이 문서는 `supabase/migrations`의 0001부터 0070까지를 운영/개발 관점에서 빠르게 이해하기 위한 설명서입니다.

기준 원칙:
1. 이미 배포된 migration 파일은 수정하지 않고 forward-only로 추가
2. 번호/파일명은 ledger(역사)로 취급
3. 설명서는 코드 변경 시 반드시 동기화

1. `0032`는 중복 번호 파일 2개가 존재하는 역사 이력 구간입니다.
2. 신규 migration 추가 시, 이 문서 마지막 버전 행을 함께 추가합니다.
3. 상태/의미 해석이 바뀌면 `docs/migration-catalog.md`와 본 문서를 함께 갱신합니다.
