## 7. 검증 방법론

1. **colaw 데이터 접근**: `fetch('https://colaw.co.kr/rescureManage/popupRescureCreditorList', {POST})` 로 각 사건의 채권자 탭 HTML을 직접 요청, `nowtotalsum` 필드 추출
2. **VS 데이터 접근**: Supabase Management API (`/v1/projects/.../database/query`)로 SQL 직접 실행
3. **비교 기준**: colaw의 `nowtotalsum` (채권 현재액 총 합계) vs VS의 `SUM(capital + interest)` per case
4. **전수 검증**: colaw 리스트 page 1 (50건) + page 2 (40건) = 90건 전체 비교 완료
