# 개인회생·개인파산 모듈 프로젝트 목표

## 최종 목표
colaw.co.kr의 개인회생·개인파산 업무 로직을 Vein Spiral에 완전히 이식.
마이그레이션된 기존 사건과 **신규 입력 사건 모두** 정확하게 동작해야 함.

## 완료 기준 (DoD: Definition of Done)

### 개인회생
1. 신규 사건 1건을 처음부터 끝까지 입력 → 저장 → 문서 출력까지 가능
2. colaw에서 마이그레이션된 기존 사건의 계산 결과가 colaw와 일치
3. 아래 7개 탭 모두 저장/로드/검증 통과
   - 신청인 / 채권자 / 재산 / 소득·생계비 / 진술서 / 변제계획 / 출력·문서
4. 7개 법원 제출 문서 정확히 생성
   - 개시신청서 / 위임장 / 채권자목록 / 재산목록 / 수입지출목록 / 진술서 / 변제계획안
5. 계산 정확도
   - 월 가용소득 = 월소득 − 생계비 − 추가생계비 − 양육비 − 회생위원 보수
   - 변제율 (총변제액 / 총채무액) 일치
   - 청산가치 보장 원칙 검증
6. 기준중위소득 60% 테이블 공식 고시와 일치 (2024/2025/2026)

### 개인파산
1. 신규 사건 1건을 처음부터 끝까지 입력 → 저장 → 문서 출력까지 가능
2. 필수 법원 양식 생성
   - 파산신청서 / 면책신청서 / 채권자목록 / 재산목록 / 수입지출목록 / 진술서
3. 면제재산 계산 로직 구현
4. 계산 결과가 colaw와 일치

## 검증 방법
- **기존 사건 (마이그레이션된 90건)**
  - 각 사건의 핵심 수치(총채무, 가용소득, 월변제액, 변제율)가 colaw와 일치
- **신규 사건 (테스트용 1건 이상)**
  - 빈 사건으로 시작 → 모든 탭 입력 → 저장 → 로드 → 문서 출력까지 오류 없이 동작

## 진행 원칙
1. 한 번에 하나씩 (개인회생 완료 → 개인파산)
2. 탭 하나 완료 후 다음 탭
3. 조사관(Sonnet) — 보고서 작성 전담
4. 실행자(Opus, CLI) — VS 코드 수정 전담
5. 모든 수정은 조사관 보고서 기반
6. 배포 후 반드시 검증 보고서로 확인
