## P0 — 치명적 오류 (법원 제출 불가)

### P0-1. 가용소득 불일치 (탭 vs 문서)
- **위치**: `document-generator.ts` (변제계획안 문서)
- **현상**: 변제계획 탭은 회생위원 보수 3% 차감 후 **447,582원/월** 사용, 문서 출력은 차감 전 **461,425원/월** 사용
- **파급**: 월변제액, 총변제액(26,854,920 vs 27,685,500), 변제율(16.4% vs 17.1%), 채권자별 배분액 전부 불일치
- **수정**: document-generator.ts에서 `availableIncome` 계산 시 `commission_rate` (3%) 차감 로직 추가

### P0-2. 대리인 정보 전체 문서 빈칸
- **위치**: 개시신청서, 위임장, 변제계획안 등 모든 문서
- **현상**: 법무법인명, 전화, 이메일 전부 빈칸. 위임장의 "을(를)" 위치에 대리인명 누락
- **수정**: `organization` 또는 별도 `attorney` 테이블에서 대리인 정보 fetch → 문서에 주입

### P0-3. 변제기간 날짜 빈칸
- **위치**: 변제계획안 문서
- **현상**: 변제 시작일/종료일이 `[ ____ ]년 [ __ ]월` 형태로 비어있음
- **수정**: `rehabilitation_plan_settings`의 `plan_start_date` / 계산된 종료일을 문서에 주입

### P0-4. 사건번호 빈칸
- **위치**: 개시신청서 헤더
- **현상**: "수원회생법원 개회 호" — 사건번호 미기입
- **수정**: 사건 데이터에서 `case_number` 가져와 문서 헤더에 바인딩

## P1 — 중요 오류 (데이터 정확도/완성도)

### P1-1. 채권자 목록 — 합계 vs 무담보 회생 채권액 불일치
- **현상**: 합계 163,600,000원 (원금+이자) vs 무담보 회생 161,850,000원 (원금만)
- **원인**: 합계 = 원금+이자, 무담보 회생 = 원금만으로 표시. 일관성 필요
- **수정**: 무담보 회생 열에도 이자 포함 합계를 표시하거나, 합계를 원금만으로 통일

### P1-2. 채권자 목록 — 채권현재액 산정기준일 빈칸
- **현상**: 상단 "채권현재액 산정기준일:" 및 각 채권자의 "부채증명서 참조(산정기준일: )" 전부 빈칸
- **수정**: 채권자 데이터에 산정기준일 필드 추가 또는 사건 등록일 기본값 사용

### P1-3. 채권자 목록 — 주소 및 연락처 빈칸
- **현상**: 5개 채권자 모두 주소/연락처 열이 비어있음
- **수정**: 채권자 테이블에 주소/연락처 필드 추가 및 문서에 바인딩

### P1-4. 재산 목록 — 명칭 빈칸
- **현상**: 재산 4,830,000원의 명칭(예: 임차보증금, 퇴직금 등)이 표시되지 않음
- **수정**: `rehabilitation_assets` 테이블의 `asset_name` 또는 `asset_type`을 명칭 열에 바인딩

### P1-5. 재산 목록 — 면제재산 설명 placeholder
- **현상**: "면제재산 결정신청 금액 (1. 설명)", "(2. 설명)" — placeholder 텍스트 그대로 노출
- **수정**: 면제재산 데이터가 없으면 해당 행 숨기기, 있으면 실제 설명 표시

### P1-6. 수입지출 목록 — 급여소득 상세 빈칸
- **현상**: 급여소득 행의 기간구분/금액/연간환산금액/압류 유무 전부 빈칸 (연수입/월평균만 하단에 표시)
- **수정**: income_settings에서 기간구분(월), 금액(2,981,000), 연간환산(35,772,000) 바인딩

### P1-7. 수입지출 목록 — 고용/자영 구분 미체크
- **현상**: 수입상황 테이블에서 자영(상호)/고용(직장명) 어디에도 체크 없음
- **수정**: 소득 유형에 따라 해당 열에 ■ 또는 ✓ 표시

### P1-8. 변제계획안 — 채권액 불일치 (변제 내용 vs 변제예정액표)
- **현상**: 신한카드 변제 내용 테이블 23,600,000원 vs 변제예정액표 23,950,000원 (이자 포함 여부 차이)
- **수정**: 동일 기준(원금+이자 또는 원금만)으로 통일

### P1-9. 진술서 — 거의 전체 빈칸
- **현상**: 경력(학력/직장/결혼), 주거상황(시작일/상세), 부채상황(사유/상세), 면책절차 — 전부 빈칸 또는 placeholder
- **수정**: 진술서 탭 입력 데이터를 문서 생성기에 매핑

## P2 — 개선 사항

### P2-1. 수입지출 목록 — 생계비 표시 로직
- **현상**: "1인 가구 중위소득 60%인 1,538,543원을 지출할 것으로 예상" 표시. 실제 사용 생계비 2,519,575원과 불일치
- **참고**: 법적 양식상 맞을 수 있으나, 실제 생계비와 다른 수치가 표시되면 혼란

### P2-2. 수입지출 목록 — 가족관계 빈칸
- **현상**: 가족관계 테이블 헤더만 있고 데이터 없음
- **수정**: 가족 데이터가 없으면 "해당 없음" 표시

### P2-3. 개시신청서 — 내용 과소
- **현상**: colaw는 주소, 직업, 재산요약, 채권요약 등 상세 정보 포함. VS는 채무자명/주민번호만 표시
- **수정**: 신청서에 필요한 추가 필드(주소, 직업, 관할법원 등) 추가

### P2-4. 위임장 — 위임사항 목록 누락
- **현상**: colaw 위임장은 위임사항(1. 개인회생절차개시의 신청, 2. 변제계획안 작성 등) 포함. VS는 위임 문구만
- **수정**: 표준 위임사항 목록 추가

### P2-5. 진술서 — 주거상황 기본값 오류
- **현상**: "① 신청인 소유의 주택"이 기본 체크되어 있으나, 임차인인 경우가 더 일반적
- **수정**: 진술서 탭의 주거 유형 데이터에 따라 동적으로 체크

### P2-6. 모달 닫기 UX 문제
- **현상**: 문서 미리보기 모달의 X 버튼/Escape 키가 간헐적으로 동작하지 않음
- **수정**: dialog close 이벤트 핸들러 점검

### P2-7. 주민번호 마스킹 차이
- **현상**: VS는 820517-******* (7자리 마스킹), colaw는 820517-***** (5자리 마스킹)
- **참고**: 법원 제출 기준에 맞춰 통일 필요
