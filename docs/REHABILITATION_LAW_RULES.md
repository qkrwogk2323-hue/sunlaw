# 개인회생 법률 규칙 (REHABILITATION_LAW_RULES.md)

> **목적**: 채무자 회생 및 파산에 관한 법률(이하 "법"), 개인회생사건 처리지침(재민 2004-4, 시행 2026.3.1), 서울회생법원 변제계획안 작성방법 가이드를 기반으로, Vein Spiral 개인회생 모듈이 **반드시** 준수해야 하는 계산 규칙을 코드 수준으로 정의한다.
>
> **법적 근거 약어**: 법 = 채무자회생법, 지침 = 재민 2004-4, 규칙 = 채무자회생규칙, 가이드 = 서울회생법원 변제계획안 작성방법

## 1. 채권 분류 체계 (Priority Classification)

### 1-1. 3단계 분류

개인회생채권은 아래 순서대로 분류하며, **변제 순위가 다르다**.

| 순위 | 분류 | 법적 근거 | 변제 조건 |
|------|------|---------|---------|
| 1 | **개인회생재단채권** (우선변제) | 법 §583 | 원리금 **전액(100%)** 우선 변제 |
| 2 | **일반의 우선권 있는 개인회생채권** | 법 §614①④, 지침 p.7 §4 | 원리금 **전액(100%)** 변제 |
| 3-a | **별제권부 채권** | 법 §614①③ | 담보 처분 → 부족분만 일반 안분 |
| 3-b | **일반 개인회생채권** (무담보) | 법 §614①④ | 안분(pro-rata) 변제 |
| 3-c | **미확정 개인회생채권** | 법 §614①④ | 일반과 동일 안분, 유보 처리 |

### 1-2. 개인회생재단채권 (법 §583) — 우선변제 대상

다음은 **재단채권**으로서 변제계획 인가 직후 **원리금 전액을 일반 채권보다 먼저** 변제해야 한다:

1. 납부기한이 도래하지 아니한 원천징수하는 조세
2. 부가가치세·개별소비세·주세 및 교통·에너지·환경세
3. 특별징수의무자가 징수하여 납부하여야 하는 지방세
4. 위 각 조세의 부과·징수의 예에 따라 부과·징수하는 교육세 및 농어촌특별세
5. 채무자의 근로자의 **임금·퇴직금 및 재해보상금**
6. 개인회생절차개시결정 전의 원인으로 생긴 채무자의 근로자의 임치금 및 신원보증금의 반환청구권 등

### 1-3. 일반의 우선권 있는 개인회생채권 (지침 §4, 가이드 p.7)

국세징수법 또는 국세징수의 예에 의하여 징수할 수 있는 청구권:
- **국세, 지방세** 등 지방자치단체의 징수금
- **관세 및 가산금**
- **건강보험료, 산업재해보상보험료** 등

> **핵심 규칙**: 이 채권들도 **전액 변제** 대상이다.

### 1-4. 코드 매핑

```
DB: has_priority_repay = true  →  priorityClass = 'tax_priority'
DB: is_secured = true          →  priorityClass = 'secured'
DB: (그 외)                     →  priorityClass = 'unsecured'
```

**[수정됨 2026-04-11]** `generatePrioritySchedule()`에서 `hasPriorityRepay=true` 채권 100% 배분 구현. `RepaymentResult`에 `priorityDebt`/`priorityInsufficient` 추가.

## 2. 변제기간 (Repayment Period)

### 2-1. 기본 규칙 (법 §611, 지침 §8)

| 구분 | 기간 | 조건 |
|------|------|------|
| **기본** | **36개월 (3년)** | 원칙. 원금+이자 전부 변제 불가능한 경우 |
| **연장** | 최대 **60개월 (5년)** | 청산가치 보장 등 법 §614①④ 충족 필요시 |
| **72개월** | **존재하지 않음** | 어떤 경우에도 불가 |

### 2-2. 변제기간 결정 순서 (지침 §8②)

1. 변제기간 동안 **가용소득의 전부**를 투입하여 우선 원금을 변제하고 잔여금으로 이자를 변제한다.
2. 5년 이내의 변제기간 동안 위 방법으로 법 §614①④(청산가치 보장)를 충족할 수 있는 때에는 그때까지를 변제기간으로 한다.
3. 법원은 채무자가 단기간을 제출한 경우 수정을 명할 수 있다.

### 2-3. 변제시작일 (지침 §7③, 가이드 p.3)

```
변제시작일 = 변제계획안 제출일 + 60~90일 내의 일정한 날
```
- 변제시작일 설정 이유: 개시신청~인가결정까지 약 3-4개월간 변제실적을 법원이 인가자료로 활용 + 채무자가 조정된 생계비에 미리 적응 (가이드 p.3 각주6)
- **[미구현]** VS에 자동계산 없음

### 2-3a. 임치기간 산입 (지침 §8④)

> "변제계획안의 인가 전에 매월 변제액을 회생위원에게 임치한 경우에는 그 **임치한 기간을 변제기간에 산입**한다."

- **[미구현]** VS에 임치기간 공제 로직 없음

### 2-3b. 비월 변제 (지침 §8⑤)

> "농업소득자, 임업소득자 등 소득이 매월 발생하지 않는 채무자는 ... **수개월 간격으로 변제**하는 것으로 ... 정할 수 있으며, 법원은 ... 이를 허가할 수 있다."

- **[미구현]** `repayment_method: '매월' | '격월' | '분기'` 필드는 존재하나 계산에 미반영

### 2-4. §8②3~5호 삭제 경위 (부칙 2018.06.07 제1693호)

2018년 이전에는 5년(60개월)이 기본 변제기간이었으나, 제1693호 부칙으로 §8②3~5호가 삭제되어 **3년(36개월)이 기본**으로 전환됨. `capital36` 디폴트의 법적 근거.
- 적용: "시행 후 최초로 신청하는 개인회생사건부터 적용" (2018.6.13 이후 신청 건)

### 2-4. 코드 매핑

```typescript
type RepayPeriodOption =
  | 'capital36'       // 기본: 3년 원금변제, 이자 면책
  | 'both36'          // 3년 원리금변제
  | 'capital60'       // 5년 원금변제 (예외)
  | 'both60'          // 5년 원리금변제
  | 'capital100_5y'   // 3~5년 내 원금 100% 변제 (5년까지 이자추가)
  | 'capital100_3y';  // 3년 내 원금 100% 변제

// DEFAULT = 'capital36' (36개월)
```

**[BUG]** 레거시 91건이 `capital60`으로 마이그레이션됨. `capital36`이 올바른 디폴트.

## 3. 가용소득 산정 (Available Income)

### 3-1. 공식 (법 §579④, 지침 §7)

```
가용소득 = 월소득 - 생계비 - 추가생계비 - 양육비 - 회생위원보수
```

### 3-2. 월소득 산정 (지침 §7①)

- 최근 1년간 직장 변동 없음 → **1년간 실제 소득 평균**
- 직장 변동 있음 → **변동 이후 실제 소득 평균**
- 영업소득자 소명자료 없음 → 임금구조기본통계조사보고서 등 통계소득

### 3-3. 생계비 (법 §579⑥, 지침 §7②, 가이드 p.3-4)

```
생계비 = 기준 중위소득 × 60%
```

- **기준 중위소득**: 국민기초생활보장법 §6에 따라 공표된 **개시신청 당시** 연도 기준
- **가구 수**: 채무자 본인 + 피부양자 수
- **60%는 권장선**, 하드코딩 금지. 사용자 자유 변경 가능
- 60% 미만 가능, 60% 초과 시 **추가생계비 항목으로 사유 기재** 필요

### 3-3a. 추가 생계비 (가이드 p.38 각주16)

서울회생법원 2021.2.15 기준, 추가 생계비로 인정되는 카테고리:
- **주거비** (월세 부담 등)
- **미성년자 자녀 교육비**
- **의료비**

> "다만 배우자의 피부양자(부양가족) 인정 여부와 추가 생계비(주거비)는 **법원(회생위원)의 조사결과에 따라 인정되지 않거나 감액**될 수 있습니다."

### 3-3b. 임차보증금 청산가치 제외 (가이드 p.39)

주택임대차보호법 §8에 따른 우선변제 범위 내 임차보증금은 압류 불가 재산 → **청산가치에서 전액 제외**.

서울특별시 기준: 보증금 3,700만원 이하 / 우선변제 임차인 범위 1억1,000만원 이하

### 3-4. 회생위원 보수 (지침 §10, 별표1, 가이드 p.4)

| 회생위원 유형 | 인가 전 보수 | 인가 후 보수 |
|------------|-----------|-----------|
| 법원사무관 (내부) | 0원 | 0원 |
| 외부 회생위원 | 15만원 (상한 30만원) | 임치액의 **0.5%** (상한 1%) |

```
월 회생위원 보수 = 월 가용소득 × 1% (소수점 이하 반올림)
   → 법원사무관이면 0원
```

### 3-4a. 회생위원 유형 자동 판별 조건 (가이드 p.5 각주8, 9)

```
IF 채무총액 ≤ 2억 AND 급여소득자:
  → 법원사무관 회생위원 선임 → 보수 = 0원
ELSE (영업소득자 OR 채무총액 > 2억):
  → 외부 회생위원 선임 → 보수 = 월 가용소득의 1% (소수점 이하 반올림)
```

여기서 **채무총액** = 채권자목록의 채권현재액(원금) + 채권현재액(이자) 합산 총합계

### 3-5. 코드 매핑

```typescript
// calculateMonthlyAvailable()
available = monthlyIncome - livingCost - extraLivingCost - childSupport
if (trusteeCommRate > 0) {
  available = Math.round(available * (1 - trusteeCommRate / 100))
}
```

## 4. 우선변제채권 100% 규칙 (Priority Repayment Rule)

### 4-1. 법적 근거

- 법 §614①: 변제계획 인가 요건
- 법 §583: 개인회생재단채권 범위
- 지침 가이드 p.6: 재단채권은 "변제계획 인가 직후 원리금 전액을 일반 개인회생채권보다 **우선하여** 변제한다"
- 지침 가이드 p.7: 일반의 우선권 있는 채권도 "우선권의 근거를 기재"하고 전액 변제

### 4-2. 계산 규칙

```
우선변제채권_총액 = Σ(has_priority_repay == true인 채권의 원금 + 이자)

ASSERT: 우선변제채권에 대한 변제액 == 우선변제채권_총액  (100%)

위반 시 → 변제계획안 인가 불가
```

### 4-3. 변제 순서 (계단식 배분)

```
Phase 1: 우선변제 단독 구간
  - 매월 변제액 전액을 우선변제채권에 배분
  - 우선변제채권 간에는 원금 비율로 안분

Phase 2: 혼합 구간 (우선변제 잔액 + 일반 시작)
  - 우선변제 잔여분 + 남은 금액으로 일반 안분 시작

Phase 3: 일반 단독 구간
  - 매월 변제액 전액을 일반 채권에 pro-rata 안분
```

### 4-4. 가용소득 충분성 검증

```
총_가용소득 = 월_가용소득 × 변제기간

IF 총_가용소득 < 우선변제채권_총액:
  → ERROR: "가용소득이 우선변제채권 총액보다 적어 변제계획 수립 불가"
  → 변제기간 연장(최대 60개월) 또는 가용소득 증액 필요
```

**[수정됨 2026-04-11]** `generatePrioritySchedule()`로 Phase 1(100% 배분) → Phase 2(일반 pro-rata) 구현. 가용소득 부족 시 `priorityInsufficient` 경고.

## 5. 청산가치보장 원칙 (Liquidation Value Guarantee)

### 5-1. 법적 근거

- 법 §614①④: "개인회생채권자의 채권이 ... 파산절차에서 배당받을 수 있는 가치보다 적지 아니한 것"
- 가이드 p.5, p.12, p.16: 청산가치와의 비교 필수

### 5-2. 정의

```
청산가치 = 재산목록 합계액 (= 파산 시 채권자가 배당받을 수 있는 가치)
현재가치 = 총변제예정(유보)액의 라이프니쯔 할인 현재가치
```

### 5-3. 판정 공식

```
IF 현재가치(총변제예정액) < 청산가치:
  → 재산처분 투입 필요
  → 변제투입예정액 = (청산가치 - 현재가치잔액) × 계수

계수 (법원사무관 회생위원):
  - 변제기한 1년 이내: × 1.3
  - 변제기한 2년 이내: × 1.5

계수 (외부 회생위원):
  - 위 금액에서 1%(소수점 이하 반올림) 차감
```

### 5-4. 라이프니쯔 현재가치 계산 (가이드 p.12, p.16)

```
현재가치(L) = 월변제예정(유보)액 합계(H) × 라이프니쯔계수
```
**반올림 규칙: 원 미만 '버림' (Math.floor)**

라이프니쯔계수 도출:
```
계수 = 적립기간 + leibniz(변제투입기간)
적립기간 = 3 (개월, 개시신청~인가 약 3-4개월)
변제투입기간 = 변제횟수 - 적립기간

leibniz(n) = (1 - (1 + 5/1200)^(-n)) / (5/1200)
  할인율: 민법 §379 연 5%, 월 복리 = 5/1200

36개월: 3 + leibniz(33) = 3 + 30.7719 = 33.7719
48개월: 3 + leibniz(45) = 3 + 40.9555 = 43.9555
60개월: 3 + leibniz(57) = 3 + 50.6433 = 53.6433
```

### 5-5. D5110 vs D5111 양식 선택

```
IF 현재가치(총변제예정액) > 청산가치:
  → D5110 (가용소득만으로 변제) 사용, 재산처분 불필요
ELSE:
  → D5111 (가용소득 + 재산처분) 사용, 처분재산 기재 필수
```

### 5-6. D5111 재산처분 투입액 공식 (가이드 p.14-17)

```
(O) 변제투입예정액 = { (J)청산가치 - (L)현재가치 } × 승수
```
**반올림 규칙: 원 미만 '올림' (Math.ceil)**

승수 조건:
- 변제기한 인가일부터 **1년 이내**: × **1.3**
- 변제기한 인가일부터 **2년 이내**: × **1.5**

외부 회생위원인 경우:
```
실제(O) = 위 산출액 × 0.99  (1% 보수 차감, 소수점 반올림)
```

재산처분 채권자별 배분:
```
(P) = (O) × { (D)해당 채권액 / (G)총 채권액 }
```
**반올림 규칙: 원 미만 '올림' (Math.ceil)**

변제기한 원문:
> "원칙적으로 '인가일부터 1년 내'로 하고 1년 내에 처분하기 곤란한 사정이 있으면 '인가일부터 2년 내'로 기재"

미처분 시:
> "위 변제기한까지 재산이 처분되지 않거나 ... 미달하는 경우, 채무자는 그 밖의 다른 재산을 처분하는 등의 방법으로 금원을 조달하여 변제투입예정액 전액을 변제기한내에 변제하여야 한다."

**[미구현]** D5111 재산처분 배분 로직은 VS에 아직 구현되지 않음.

## 6. 별제권 처리 (Secured Claims)

### 6-1. 담보물건 평가

```
청산가치(담보) = 시장가치 × 감정비율(통상 70%)
```

### 6-2. 순위별 배분 (동일 담보물건)

같은 담보물건에 복수 채권자가 근저당/질권을 가진 경우:

```
FOR 순위 = 1순위 부터:
  잔여_담보가치 = 청산가치(담보) - 선순위_배분_합계
  해당_채권자_배분 = MIN(잔여_담보가치, 채권현재액)
  해당_채권자_부족분 = MAX(0, 채권현재액 - 해당_채권자_배분)
  → 부족분은 일반 무담보 채권으로 전환
```

### 6-3. 별제권부 채권의 변제계획 기재 (가이드 p.7-8 §5)

**가. 채권의 내용 (표 형식):**
- ① 채권현재액(원금)
- ② 채권현재액(이자)
- ③ 별제권행사 등으로 변제가 예상되는 채권액
- ④ 별제권행사 등으로도 변제받을 수 없을 채권액
- 별제권 등의 내용 및 목적물: 설정일, 원금, 이자율, 채권최고액, 환가예상액, 목적물 소재

**나. 변제방법 (가이드 원문):**
> **(1)** 별제권행사로 변제 예상 채권액(③)은 별제권행사에 의해 변제하고 가용소득/재산처분 변제대상에서 **제외**
> **(2)** 별제권행사로도 변제 불가 채권액(④)은 **미확정채권으로 보아 유보** → §7 방법으로 변제
> **(3)** ④ 초과 확정 시 → 변제계획안 **변경 절차**로 변제 가능

### 6-4. 변제 배분 규칙

```
별제권부 채권 변제액 = MAX(청산가치(담보배분액), 일반안분변제액)
```

> **[원본 근거 미확인]** 이 MAX 규칙은 법률 일반론에서 도출된 것으로, 처리지침(재민 2004-4) 및 서울회생법원 가이드 PDF에 **명시적으로 기재되어 있지 않다**. 가이드 §5는 별제권 채권을 가용소득/재산처분 변제대상에서 "제외"한다고만 명시. 실무 확인 필요.

### 6-5. DB 스키마 매핑

```sql
-- rehabilitation_secured_properties (별제권 담보물건)
-- ※ rehabilitation_properties와 별개 테이블
id, case_id, property_type, description, market_value,
valuation_rate (default 70), note, sort_order, lifecycle_status

-- rehabilitation_creditors (별제권 관련 컬럼)
is_secured, secured_property_id → FK → rehabilitation_secured_properties.id
secured_collateral_value,  -- 담보평가액 (= market_value × valuation_rate / 100)
lien_type, lien_priority, max_claim_amount, remaining_unsecured
```

## 7. 대위변제 (Subrogation) 처리

> **[법률 일반론 기반]** 아래 계산 공식은 민법상 대위변제 법리에서 도출된 것이다. 처리지침(재민 2004-4)에는 D5130 양식명만 언급되며 **구체적 계산 공식은 명시되어 있지 않다**. 실무 관행에 기반한 정리이므로 법원별 차이가 있을 수 있다.

### 7-1. 일부대위변제 (법원서식 D5130)

보증인이 채무 일부를 변제한 경우:

```
원채권자_잔여채권 = 원금잔액 + 이자잔액 - 보증인변제액
보증인_구상채권 = 보증인변제액 (일반 무담보 채권으로 별도 등재)

→ 채권자목록에 2행:
  1. 원채권자 (감소된 금액)
  2. 보증인 (구상채권)
```

### 7-2. 전액대위변제

보증인이 원금 전액을 변제한 경우:

```
원채권자_잔여채권 = 이자만 남음 (원금 0)
보증인_구상채권 = 원금 전액 (일반 무담보)

→ 채권자목록에 2행:
  1. 원채권자 (이자만)
  2. 보증인 (구상채권 = 원금)
```

### 7-3. 코드 매핑

```
DB: guarantor_name, guarantor_amount, guarantor_text
→ 보증인 변제 정보 (UI 입력·표시만, 계산 로직에서 사용 안 됨)
→ D5130 서식: 코드 미구현
→ secured-allocation.ts: 보증인/대위변제 로직 없음
```

## 8. 안분 변제 계산 (Pro-rata Distribution)

### 8-1. 기본 공식 (가이드 p.11, p.14)

```
(E) 채권자별_월변제예정액 = (A)월_실제_가용소득 × { (D)해당채권 확정채권액 / (G)확정채권액 총계 }
```

- 확정채권과 미확정채권은 **별도 산정** 후 합산
- **반올림 규칙: 원 미만 '올림' (Math.ceil)**
- 올림 결과, (H)월 변제예정(유보)액 합계는 (A)월 실제 가용소득보다 **약간 많아짐** (정상)
- 가이드 원문: "원 미만은 '올림'으로 처리하여 ... 이 월 변제예정(유보)액은 이미 기재한 '월 실제 가용소득'보다 약간 더 많은 금액이 될 것입니다."

### 8-2. 안분 기준

```
일반채권:  capital (원금) 기준 안분이 원칙 (가이드: "원금의 액수를 기준으로 안분")
```

단, 변제액이 커서 원금 외에 개시결정일 전날까지의 이자·지연손해금도 변제 가능한 경우:
```
안분 기준 = 원금 + 이자 + 지연손해금 합계액
```

### 8-3. 재산처분 투입 시 (가이드 p.15)

```
(P) 재산처분_채권자별_총변제예정액 = (O)실제_변제투입예정액 × { (D)해당채권 / (G)총채권액 }
```
**반올림 규칙: 원 미만 '올림' (Math.ceil)**

### 8-4. 반올림/올림/버림 규칙 총정리 (가이드 p.12-17)

| 항목 | 처리 방법 | 적용 위치 | 코드 함수 |
|------|----------|----------|----------|
| **(E)** 월 변제예정(유보)액 (채권자별) | **올림** (원 미만) | `schedule-generator.ts` | `Math.ceil` |
| **(P)** 재산처분 총변제예정(유보)액 (채권자별) | **올림** (원 미만) | 미구현 (D5111) | `Math.ceil` |
| **(O)** 변제투입예정액 | **올림** (원 미만) | 미구현 (D5111) | `Math.ceil` |
| **(L)** 현재가치 (라이프니쯔) | **버림** (원 미만) | `leibniz.ts` | `Math.floor` |
| ④ 월 회생위원 보수 | **반올림** (소수점) | `repayment-calculator.ts` | `Math.round` |
| 재산처분 회생위원 보수 | **반올림** (소수점) | 미구현 | `Math.round` |
| 변제율(%) | **반올림** (소수점) | 표시 시 | `Math.round` |

## 9. 변제예정액표 작성 규칙 (Repayment Schedule Table)

### 9-1. D5110 양식 전 셀 산출식 (가용소득만)

```
③ = 월평균가용소득 (수입-생계비-추가생계비)
④ = 법원사무관→0, 외부→③×1% (반올림)
(A) = ③ - ④
(B) = 변제기간 개월수
(C) = (A) × (B)

FOR 각 채권자:
  (D) = 확정채권액(원금) [미확정은 별도행]
  (E) = (A) × { (D) / (G) }    ← 올림
  (F) = (E) × (B)

(G) = Σ(D)  [확정+미확정 총계]
(H) = Σ(E)  [(H) > (A) 정상]
(I) = Σ(F)

변제율 = (I)/(G) × 100  ← 소수점 반올림

(J) = 청산가치 (재산목록)
(K) = (I)
(L) = (H) × 라이프니쯔계수  ← 버림
```

### 9-2. D5111 양식 추가 셀 (가용소득 + 재산처분)

D5110의 (A)~(L)에 추가:

```
(O) = { (J) - (L) } × 승수    ← 올림
  승수: 1년이내=1.3, 2년이내=1.5
  외부 회생위원: (O) = 위 × 0.99 (1% 보수 차감, 반올림)

FOR 각 채권자:
  (P) = (O) × { (D) / (G) }    ← 올림

(Q) = Σ(P)
(M) = (Q)
(N) = 기재 불필요
```

### 9-3. 적립기간 / 변제투입기간 (가이드 p.10, p.13)

```
적립기간 = 3 (개월) — 개시신청~인가 약 3-4개월 적립
변제투입기간 = 변제횟수 - 적립기간 (예: 36 - 3 = 33)
```

### 9-4. 회차별 단계변제 (가이드 작성례1 사례2, p.24-26)

원금 전부 + 이자 일부 변제가 가능한 경우:
```
Phase 1 (제1~N회): 원금만 안분
  (E) = 가용소득 × { 원금 / 원금총계 } [올림]

Phase 2 (제N+1회): 원금 잔액 + 이자 배분 시작
  원금잔액 = 원금 - Phase1 총변제
  이자배분 가용액 = 가용소득 - Σ(원금잔액)
  이자분 = 이자배분가용액 × { 해당이자 / 이자총계 } [올림]

Phase 3 (제N+2~36회): 이자만 안분
  (E) = 가용소득 × { 이자 / 이자총계 } [올림]
```

**[미구현]** VS에 회차별 단계변제 로직 없음. 현재는 단일 비율 배분만 지원.

## 부록 A: 현재 코드의 버그 목록

### A-1. [P0] 우선변제 100% 미준수

- **위치**: `schedule-generator.ts` `generateTieredSchedule()`
- **증상**: `has_priority_repay=true`인 국세청 채무가 17.6%만 변제됨
- **원인**: `hasPriorityRepay` 필드를 계산 로직에서 전혀 읽지 않음. `tieredTaxPriority` 모드에서도 100% 보장 검증 없음
- **법적 위반**: 법 §583, §614①, 지침 가이드 p.6

### A-2. [P0] 청산가치 0원 표시

- **위치**: 프론트엔드 소득·생계비 탭
- **증상**: DB에 재산 합계(43.8M~88.8M)가 있으나 UI에 0원으로 표시
- **원인**: 청산가치 데이터가 프론트엔드로 전달되지 않음

### A-3. [P1] 변제기간 디폴트 오류

- **위치**: 마이그레이션 데이터 91건
- **증상**: `repay_period_option = 'capital60'` (60개월)
- **원인**: 마이그레이션 시 디폴트를 잘못 설정
- **수정**: 디폴트를 `'capital36'` (36개월)로 정정

### A-4. [P1] 채권자 번호 재정렬 미구현

- **위치**: UI 채권자 관리
- **증상**: 우선변제 채권을 1번에 삽입해도 기존 채권 번호가 밀리지 않음
- **필요**: 분류 순서(재단→우선→별제→일반→미확정)에 따른 자동 정렬

### A-5. [P1] capitalOnly 모드 안분 분모 의문

- **위치**: `schedule-generator.ts` line 49
- **증상**: `capitalOnly=true`인데 `totalDebt`에 `(capitalOnly ? 0 : interest)` 적용. 원금만 변제 모드에서 안분 분모가 원금만인지, 원리금 합계인지 불명확
- **법적 정답**: "원금의 액수를 기준으로 안분" (가이드 p.8)

### A-6. [P2] gross_salary 동기화

- **위치**: income_settings DB ↔ 프론트엔드 폼
- **증상**: DB 값과 폼 표시 값이 불일치

### A-7. [P2] 아코디언 렌더링

- **위치**: 채권자 상세 폼
- **증상**: 펼치면 거대한 빈 공간 발생, 스크롤 불가
