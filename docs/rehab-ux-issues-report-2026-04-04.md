### 2. [Resolved] 변제기간 NaN → 빈칸 fallback

- **수정 전**: `[ NaN ]년 [ NaN ]월 [ NaN ]일`
- **수정 후**: `[ ____ ]년 [ __ ]월 [ __ ]일` (application_date 미입력 시 빈칸 처리)
- **원인**: `new Date('')` → NaN 방어 처리 추가, 신청일 기반 자동 산출 로직 적용

### 3. [Resolved] 채권자별 변제액 100% 전액변제 → 가용소득 기반 안분 계산

- **수정 후**: 신한카드 67,282원/월 (가용소득 × 채무비율 안분)
- **검증**: 월 총 변제액 합계 461,422원 ≈ 가용소득 461,425원 (3원 반올림 차이) ✅

### 4. [Resolved] 채무자 이름 빈칸 → 정상 출력

## 미해결 이슈

### 5. [High] 변제계획 탭 — DB 영속성 없음

**현상**: 변제기간 옵션(3년/5년), 변제 방식(원금우선/균등) 선택이 페이지 새로고침 시 초기화됨. DB의 `repay_months`는 60(소득탭 기본값)인데, 변제계획 탭에서 36개월로 변경해도 DB에 반영되지 않음.

**영향**: 문서 출력 시 변제계획 탭의 계산 결과가 아닌 DB 기본값(60개월)이 사용됨. 탭에서는 36개월 × 447,582원이지만, 문서에서는 60개월 × 461,425원으로 나옴.

**원인**: `rehab-plan-tab.tsx`가 순수 클라이언트 컴포넌트로, 저장 버튼과 server action이 없음.

**수정 방향**: 변제계획 탭에 저장 기능 추가. `rehabilitation_income_settings`의 `repay_months`, `repay_period_option`, `repay_rate_display` 등을 업데이트하는 server action 필요.

---

### 6. [High] 변제기간 시작/종료일 빈칸

**현상**: 변제계획안 문서에서 `[ ____ ]년 [ __ ]월 [ __ ]일부터 ... 까지 [ 60 ]개월간`으로 날짜만 빈칸.

**원인**: DB `rehabilitation_applications.application_date`와 `repayment_start_date`가 모두 null. 신청인 탭에서 해당 필드 입력 UI가 있으나 데이터를 입력하지 않았음.

**수정 방향**: 데이터 입력 누락이므로 UX 개선은 (A) 신청일 필수 입력 처리, (B) 출력 전 검증에서 누락 필드 안내.

---

### 7. [Medium] 대리인 정보 비어있음

**현상**: 개시신청서·변제계획안에서 법무법인명, 변호사명이 빈칸. "대 리 인: / 변호사"로만 표시.

**원인**: DB `rehabilitation_applications.agent_name`과 `agent_type`이 null. 데이터 미입력 상태.

**수정 방향**: 데이터 입력 누락이므로 (A) 출력 전 누락 필드 경고, (B) 대리인 없는 경우 "대리인" 섹션 자체를 생략하는 조건 분기.

---

### 8. [Medium] 소득/생계비 탭 — 저장 버튼 스크롤 불가

**현상**: 저장 버튼이 뷰포트 아래에 위치하는데, 사이드바가 메인 콘텐츠 영역의 스크롤 이벤트를 가로채서 버튼까지 스크롤 불가.

**재현 조건**: 1400×868px 이하 화면에서 소득/생계비 탭 진입 시.

**수정 방향**: 메인 콘텐츠 영역에 독립적인 스크롤 컨테이너 적용, 또는 저장 버튼을 화면 하단에 sticky 배치.

---

### 9. [Medium] 변제예정액표 — 산정내역·변제율·청산가치 비교 빈칸

**수정 방향**: 가용소득, 총 변제액, 변제율(총변제액/총채무액), 청산가치(재산 목록에서 가져옴)를 자동 계산하여 텍스트 생성.

### 소득/생계비 산출 방식 차이 (중요)

colaw는 변제계획안 문서에서 "월 평균 수입 2,000,000원"과 "생계비 1,538,543원"을 사용하는데, 이는 Vein Spiral의 net_salary(2,981,000원)와 living_cost(2,519,575원)와 다름.

그러나 **가용소득은 거의 동일**(461,457 vs 461,425, 32원 차이)하므로, colaw가 문서용 표기에서 별도의 "월 평균 수입" 필드를 사용하는 것으로 추정.
Vein Spiral의 net_salary − living_cost = 461,425원과 colaw의 461,457원은 실질적으로 동일한 계산 결과.

**핵심 원인**: colaw의 "수입지출/변제기간" 탭에서 별도 입력한 문서용 수입/생계비 필드가 있으며, 이는 소득탭의 net_salary와 별개. Vein Spiral은 이 구분이 없어 소득탭 데이터를 직접 사용.
